# 个人作品集主站 — 设计文档

日期：2026-06-20
参考站：https://shenxianovo.com/ （主站 Next.js 作品集 + `blog` 子域名 Astro/Fuwari 博客）

## 目标

搭建一个属于本人的开发者个人作品集主站，观感参考 shenxianovo.com：简洁、现代、暗色模式、双语。本期**只做主站**，博客作为后续阶段（届时用 Astro + Fuwari，拆 `blog` 子域名）。

## 技术栈决策

- **Astro**（纯静态，默认零 JS）— 2026 年内容型个人站的主流现代选择：Lighthouse 易满分、Cloudflare Pages 免费托管、Markdown/MDX 与 View Transitions 原生支持。
- **TailwindCSS** — 样式。
- **TypeScript** — 类型与数据文件。
- 不使用 Next.js：纯个人站无需 SSR/服务器算力，Astro 更轻更省成本。

## 范围

### 本期做
- 单个 Astro 项目，输出纯静态站点。
- 首页（单页滚动）+ About 页。
- 暗色 / 亮色模式切换。
- 中英双语切换。
- 所有个人内容集中在一个数据/配置文件，改一处即可更新全站。
- 部署到 Cloudflare Pages（默认）。

### 本期不做（YAGNI）
- 博客、后端、数据库、CMS。
- 子域名拆分（仅在结构上预留扩展位）。

## 页面与区块

**首页（单页 + 锚点）**
1. 导航栏：Logo/头像、Home / About / Projects（锚点）、Blog（占位/禁用）、暗色切换、语言切换。
2. Hero：tagline + 头像/插画 + 两个 CTA（查看项目 / 联系我）。
3. 状态卡片：当前状态（如「开放合作中」「在 XX 学习/工作」）。
4. 能力/特点卡片：4 张卡（整洁代码 / 以用户为中心 / 性能优化 / 持续学习），文案可改。
5. 项目区：项目卡片网格（标题、描述、技术标签、链接）。
6. 页脚：版权 + 社交链接（GitHub / 邮箱等）。

**About 页**：稍详细的个人介绍。

## 数据 / 内容架构

- 单一来源：`src/data/site.ts`（或 `site.config.ts`）集中存放：
  - 个人信息：名字/昵称、tagline、简介、头像路径。
  - 状态信息。
  - 能力卡片数组。
  - 项目数组（标题、描述、技术标签、链接）。
  - 社交链接数组。
  - 双语文案（zh / en 对照）。
- **本期所有内容使用占位值**，用户后续替换。

## 组件边界

- `Nav` — 导航 + 主题/语言切换。仅依赖站点配置与当前语言。
- `Hero` — 展示 tagline / CTA。依赖配置。
- `StatusCards` — 状态展示。依赖配置数组。
- `FeatureCards` — 能力卡片。依赖配置数组。
- `Projects` — 项目网格。依赖项目数组。
- `Footer` — 版权 + 社交。依赖社交链接数组。
- `LangToggle` / `ThemeToggle` — 客户端小交互（Astro island / 轻量脚本）。
- 每个组件只读自己那部分配置，可独立理解与替换。

## 部署

- 平台：**Cloudflare Pages**（纯静态、免费、国内访问相对友好）。
- 流程：本地 `npm run dev` 预览 → 确认 → 构建 → 部署。
- 域名：暂用平台免费二级域名；用户有自有域名后绑定，并预留 `blog` 子域名扩展。

## 验收标准

- `npm run dev` 本地可正常预览，页面无报错。
- 首页含上述全部区块，暗色/亮色、双语切换可用。
- 内容全部来自集中数据文件，替换占位即可改全站。
- 构建产物为纯静态，可部署到 Cloudflare Pages。
- 响应式：移动端 / 桌面端均正常。
